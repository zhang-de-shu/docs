# Qwen3.5 技术原理详解

> 发布方：阿里巴巴通义千问（Qwen）团队。**Qwen3.5 开源模型家族**于 2026 年初（除夕夜/2 月）正式开源，覆盖 **0.8B 端侧小模型 → 397B 旗舰 MoE**，共 8 个尺寸；并有原生多模态版 **Qwen3.5-Omni**。这是当前可下载权重、且架构已公开的最新一代 Qwen 开源模型（注：同期的 Qwen3.7-Max 属闭源旗舰 API，本文不以其为主体）。
> 本文重点讲**技术原理（机制怎么做的）**，而非罗列特性。核心三块：**Gated DeltaNet 线性注意力**、**3:1 混合注意力架构**、**稀疏 MoE**。

---

## 0. 先看全景：Qwen3.5 改了什么、为什么改

初代 Qwen3（2025）已经做到双模推理、119 语言、Dense+MoE 全系列。但它的注意力仍是**标准全注意力**，长上下文下逃不开 O(N²) 的算力与 KV cache 膨胀。Qwen3.5 这一代的主线是：

> **在不牺牲质量的前提下，用"线性注意力 + 全注意力混合"把长上下文推理的成本压到工业可用，并铺满从端侧到旗舰的全尺寸。**

| 维度 | 初代 Qwen3（2025） | Qwen3.5（2026） | 解决的问题 |
|------|-------------------|-----------------|-----------|
| 注意力 | 标准全注意力 | **Gated DeltaNet 线性注意力 + 全注意力，3:1 混合** | 长上下文 O(N²) 成本 |
| 记忆机制 | KV cache 全量 | DeltaNet 状态 + 门控遗忘，推理 **O(1)** | KV cache 膨胀 |
| 前馈层 | Dense / MoE | 稀疏 **MoE**（旗舰 397B） | 大而不贵 |
| 尺寸覆盖 | 多尺寸 | **0.8B → 397B** 全场景 + Omni 多模态 | 端到云全覆盖 |

**关键能力**：旗舰 397B MoE；支持 **262K 长文本**；Qwen3.5-Omni 在 215 项音视频基准拿到 SOTA。整套架构的落点就是"既快又省又不掉质量"。

---

## 1. Gated DeltaNet 线性注意力（核心改进）

这是 Qwen3.5 最关键的一处，也是和初代最大的不同。要理解它，先理解标准注意力和纯线性注意力各自的毛病。

### 1.1 两种极端的毛病

- **标准全注意力**：每个 token 和所有历史 token 算相关性，质量高，但 O(N²) 计算 + KV cache 随长度线性膨胀，长上下文太贵；
- **纯线性注意力（如早期 Linear Attention）**：把注意力近似成一个可递推的"状态"，推理是 O(1)（像 RNN 一样只维护一个固定大小的状态），很省，但**记忆管理粗糙**——写进状态的旧信息**没法快速清除**，过时信息会污染当前判断，长程精确度差。

Qwen3.5 用 **Gated DeltaNet（门控增量网络）** 把两者的优点合起来。它是 DeltaNet 的升级版，融合了 RNN 的状态更新逻辑 + 门控设计，核心是**两个互补的机制**：

### 1.2 机制一：Delta Rule（增量规则）——精确地"改写记忆"

DeltaNet 维护一个固定大小的**记忆状态（state）**，新 token 来时不是简单覆盖，而是按 **Delta Rule（增量/差值规则）** 更新：

> **只把"新信息与旧记忆的差值（delta）"写进状态——该修正的修正、该补充的补充，而不是粗暴地整体覆盖。**

这样做的好处：记忆更新是"精修"而非"重写"，**长期记忆的精确度更高**——这正是纯线性注意力欠缺的。形象地说，Delta Rule 让模型像"在已有笔记上做精准批注"，而不是"每次都撕掉重写"。

### 1.3 机制二:门控(Gating)——快速地"清除过时信息"

光有 Delta Rule 还不够:它擅长"写得准",却**不擅长"忘得快"**。Qwen3.5 加上**门控机制**:

> **用一个可学习的门(gate)控制状态的衰减/遗忘——判断哪些旧记忆已经过时,主动快速清除,给新信息腾出空间。**

> **关键洞察:门控(Gating)与 Delta 规则在记忆管理中是互补的——Delta Rule 负责"精确写入",门控负责"快速遗忘"。** 两者结合,正好补上纯 DeltaNet"缺乏快速清除过时信息能力"的短板。这也是 Gated DeltaNet 相比原始 DeltaNet 的核心升级。

### 1.4 机制三:卷积增强局部性 + O(1) 推理

Gated DeltaNet 层还用**卷积**增强对局部(邻近 token)模式的捕捉。最终整层的设计思想可以概括为:

> **卷积增强局部性 + Delta Rule 增强长期记忆精确度 + 门控控制信息流(快速遗忘) + 推理时 O(1) 复杂度。**

O(1) 是关键工程收益:线性注意力层推理时只维护一个**固定大小的状态**,不像全注意力要存全量 KV cache——**显存不随上下文增长**,这是 Qwen3.5 能在工业级大模型上吃下长上下文的根本。

> **本质:Gated DeltaNet = 把 RNN 的"O(1) 固定状态"和 Transformer 的"精确记忆"调和到一起——用 Delta Rule 写得准、用门控忘得快、用卷积抓局部,既省(O(1))又不掉长程质量。**

## 2. 3:1 混合注意力架构(为什么不全用线性注意力)

Gated DeltaNet 再好,也仍是"近似",在需要**精确全局检索**的场景(如大海捞针、精确引用)上不如全注意力。所以 Qwen3.5 **不是纯线性**,而是**混合**。

### 2.1 机制:线性注意力层与全注意力层 3:1 交替

> **每 3 层 Gated DeltaNet 线性注意力,配 1 层标准全注意力,按 3:1 的比例在网络里交替堆叠。**

```
Layer 1: Gated DeltaNet (线性, O(1), 省)
Layer 2: Gated DeltaNet (线性)
Layer 3: Gated DeltaNet (线性)
Layer 4: Full Attention (全注意力, 精确全局)
Layer 5: Gated DeltaNet
Layer 6: Gated DeltaNet
Layer 7: Gated DeltaNet
Layer 8: Full Attention
...（3:1 循环）
```

设计道理:
- **3 层线性注意力**承担绝大部分计算,以 O(1) 状态高效处理长序列、抓局部和长期记忆——**把成本压下来**;
- **1 层全注意力**周期性地"兜底",做一次精确的全局交互,把线性层可能漏掉的精确长程依赖补回来——**把质量保住**;
- 3:1 是"省"与"准"权衡的工程甜点:绝大多数层省钱,少数层保质量。

> **本质:用"多数线性层降本 + 少数全注意力层保质量"的混合配比,在长上下文效率和检索精度之间取得平衡。** 这和 DeepSeek-V4 的 CSA/HCA 交错、是同一种"分工混合"哲学的不同实现。

---

## 3. 稀疏 MoE(大而不贵)

Qwen3.5 旗舰 **397B** 用稀疏 MoE:前馈层拆成多专家,每 token 只激活少数,"总参大、单 token 算得少"。这一代延续了 Qwen 的 MoE 路线(细粒度专家 + 稀疏激活),与混合注意力叠加:

- **注意力侧**:Gated DeltaNet 把"长度维度"的成本压成 O(1);
- **前馈侧**:MoE 把"参数维度"的成本压成稀疏激活;

两者正交叠加,共同实现"397B 的容量、却只付一小部分推理代价"。再配合 **0.8B → 397B 全尺寸**,同一套架构思想从端侧贯穿到云端旗舰。

---

## 4. 一页总结

```
主线:长上下文不掉质量地降本 + 全尺寸覆盖
   │
   ├── 注意力(核心):Gated DeltaNet 线性注意力
   │     ├ Delta Rule → 精确写入记忆(增量修正,不粗暴覆盖)
   │     ├ 门控 Gating → 快速遗忘过时信息(补DeltaNet短板)
   │     ├ 卷积 → 增强局部性
   │     └ O(1) 推理 → 固定状态,显存不随上下文增长
   │
   ├── 混合架构:线性 : 全注意力 = 3 : 1 交替
   │     多数线性层降本 + 少数全注意力层保精确全局检索
   │
   ├── 前馈:稀疏 MoE(旗舰397B) → 参数维度也稀疏,大而不贵
   │
   └── 覆盖:0.8B端侧 → 397B旗舰 + Qwen3.5-Omni多模态(215项音视频SOTA)
```

**一句话**:Qwen3.5 不靠纯堆参数,而靠 **"Gated DeltaNet 把记忆做成可精修可快忘的 O(1) 状态 + 3:1 混合保住全局精度 + MoE 稀疏激活"** 三套机制,把"长上下文 + 全尺寸 + 多模态"的开源模型做到了又省又准。
