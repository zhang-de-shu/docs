# GLM-5.1 技术原理详解

> 发布方：智谱 AI（Z.ai）。GLM-5.1 于 2026-04-08 发布并开源，是 GLM-5 系列的迭代版本，定位"复杂工程环境中可自主探索、持续改进、稳定交付"的高价值 Agentic Coding 模型。
> 本文重点讲**技术原理（机制怎么做的）**，而非罗列特性。核心四块：**自回归填空预训练**、**DSA 稀疏注意力**、**MoE 架构**、**异步 Agent 强化学习（Slime）**。

---

## 0. 先看全景：GLM-5.1 用四套机制解决四个矛盾

GLM-5.1 的设计可以用"四个矛盾、四套解法"来概括，这是理解全篇的主线：

| 矛盾 | 解法 | 作用 |
|------|------|------|
| 理解与生成不可兼得 | **自回归填空（autoregressive blank infilling）** | 一套目标同时覆盖 NLU + 有条件/无条件生成 |
| 模型越大、推理越贵 | **MoE + DSA 稀疏注意力** | 砍长上下文算力，长上下文能力不掉 |
| 训练与实战脱节 | **异步 Agent RL（代号 Slime）** | 让模型在真实多步任务里成长 |
| 算力受限于单一硬件 | **国产芯片全栈适配** | 昇腾/摩尔线程/海光/寒武纪/昆仑芯/沐曦/燧原 |

**关键能力指标**：754B（7540 亿）参数的 MoE 架构；在开源模型中首次实现 **1700 步连续工具调用、8 小时自主工作**；核心工程能力对齐 Claude Opus 4.6；在昇腾上实现 **Layer 级 MoE 绝对均衡**，系统级优化使整体吞吐提升约 **30%**。

## 1. 自回归填空预训练（GLM 的"地基"机制）

要理解 GLM 系列为什么和 GPT/BERT 都不一样，必须从它的预训练目标讲起——这是 GLM 从初代起就坚持的招牌，5.1 仍以此为底座。

### 1.1 问题：三类任务的目标互相打架

传统预训练有三条路线，各有短板：
- **自编码（BERT）**：双向看上下文，强在"理解（NLU）"，但不擅长"生成"；
- **自回归（GPT）**：从左到右逐词生成，强在"生成"，但只能单向看，理解类任务吃亏；
- **编码器-解码器（T5）**：能兼顾，但结构重、参数利用率低。

没有一个框架能用**同一套目标**把"理解 + 有条件生成 + 无条件生成"都做好。

### 1.2 机制：把"挖空 + 自回归补全"合二为一

GLM 的解法叫**自回归填空（autoregressive blank infilling）**，原理一句话：

> **在输入文本里随机挖掉若干连续片段（span），然后让模型用自回归的方式，把这些被挖掉的片段一个一个"续写"回来。**

具体怎么做：
1. 给定输入 `x = [x1, ..., xn]`，**采样多个连续片段** `{s1, ..., sm}`（每个片段是一串连续 token）；
2. 把这些片段从原文挖走，留下带"空位"的文本（Part A）；被挖走的片段集中放到后面（Part B）；
3. 模型**双向地看 Part A**（保留了 BERT 式的理解能力），同时**自回归地生成 Part B 里每个空位的内容**（保留了 GPT 式的生成能力）。

这样的妙处：
- **挖小片段** → 偏"理解/完形填空"，训练出 NLU 能力；
- **挖大片段甚至整段** → 偏"长文生成"，训练出生成能力；
- 通过**调节挖空的长度和比例**，一套目标就能在"理解↔生成"之间自由滑动。

> **本质：用"挖空+自回归补全"这一个目标，统一了理解与生成，避免了为不同任务训不同模型。** 这是 GLM 区别于 GPT/BERT 的根本机制，也是它在 Coding（既要读懂上下文、又要生成代码）这类"理解+生成强耦合"任务上有底子的原因。

## 2. DSA 稀疏注意力（解决"长上下文太贵"）

这是 GLM-5/5.1 在效率上的核心改进，与 DeepSeek 同源思路（DeepSeek 也用 DSA 一类稀疏注意力）。

### 2.1 问题根源

标准全连接注意力里，每个 token 要和前面**所有** token 算相关性，计算量随序列长度近似 O(N²)。长文本（20 万字级）下这部分开销爆炸式增长，是"模型越大、上下文越长、推理越贵"的主因。

### 2.2 机制：只和"最重要"的少数 token 对话

DSA（Dynamic/Deformable Sparse Attention，稀疏注意力）的核心思路一句话：

> **不需要每个词都和所有词对话，只让它和"最重要"的那些词交流就够了；谁重要，由模型自己根据内容动态判断。**

关键观察与做法：
- **冗余假设**：研究发现长文本中约 **90% 的"词对词"关联本来就是冗余的、可丢弃的**；
- **动态筛选**：模型对每个 token 智能挑出真正相关的少数关键 token，**只在这部分上做注意力计算**，其余直接跳过；
- 形象比喻：DSA 像一个高效的会议主持人，能在 20 万字材料里**快速锁定关键段落**，把无关内容的算力省掉。

**效果**：把注意力计算量**砍掉 1/2 到 2/3**，而理解质量几乎无损。这就是"成本大幅打下来、长上下文能力一点没丢"的来源。

### 2.3 与 DeepSeek-V4 的对照（帮助理解定位）

| | GLM-5.1 DSA | DeepSeek-V4 CSA+HCA |
|--|-------------|---------------------|
| 思路 | 动态筛关键 token，丢冗余关联 | CSA(压缩+稀疏选 top-1024) ⊗ HCA(128×压缩 dense) 逐层交错 |
| 共性 | 都是"按需精算"，不为全局付 O(N²) | 同 |
| 风格 | 单一动态稀疏机制，简洁 | 双注意力交错，分工更细 |

两家都收敛到了同一个判断：**长上下文里大部分注意力是浪费的，关键是把算力花在真正相关的 token 上。**

## 3. MoE 架构（稀疏激活，大而不贵）

### 3.1 基本机制

GLM-5.1 是 **754B 总参的 MoE**：前馈层拆成很多"专家"子网络，每个 token 只激活其中一小部分专家，于是"总参数很大、单 token 实际计算量小"，兼顾容量与成本。这与 DeepSeek 等主流稀疏大模型同构。

### 3.2 工程亮点：昇腾上的 Layer 级 MoE 绝对均衡

MoE 的老问题是**负载不均**——专家被路由得忽多忽少，导致硬件利用率低、出现计算热点。GLM-5.1 与华为云/昇腾深度协同，在昇腾算力上实现了 **Layer 级 MoE 绝对均衡**：

> 在每一层，让 token 在专家间的分配做到（接近）绝对均匀，从而吃满每张卡的算力、消除热点等待。

配合系统级优化，整体**推理吞吐提升约 30%**。这是"算法均衡"和"硬件适配"协同优化的典型——不只是算法层面追求均衡，而是和具体芯片的并行方式绑定调优。

## 4. 异步 Agent 强化学习（代号 Slime，解决"训练与实战脱节"）

这是 GLM-5.1 能做到"1700 步工具调用、8 小时自主"的关键，也是它从"会补全代码"走向"能干完整软件工程"的核心。

### 4.1 问题：传统 RL 训不出长程 Agent

要让模型成为能自主跑多步任务的 Agent，得用强化学习在"真实任务交互"中训练。但传统 RL 有个效率死结：
- **生成（rollout）和训练（update）是同步耦合的**——必须等模型把一整条多步任务轨迹跑完（可能几十上百步、耗时很长），才能拿这条轨迹去更新参数；
- Agent 任务轨迹又长又慢，于是**大量算力在"等轨迹跑完"上空转**，训练效率极低。

### 4.2 机制：把"生成"和"训练"解耦，异步流水

Slime 框架的核心思路一句话：

> **把"生成轨迹"和"用轨迹训练"两件事解耦、异步并行——一边不停地用当前模型在环境里跑任务产生轨迹，另一边不停地消费已产生的轨迹更新模型，两者不再互相等待。**

带来的改变：
- **生成端**：多个 Agent 实例持续与环境（代码库、工具、测试）交互，源源不断产出多步轨迹；
- **训练端**：从轨迹缓冲里持续取数据更新参数，不必等某条长轨迹收尾；
- 配合**独创的异步智能体 RL 算法**处理"用稍旧模型产生的轨迹来更新新模型"带来的偏差问题（异步必然引入数据新鲜度问题，算法层面要做校正）。

> **本质：用"生成/训练解耦 + 异步流水"把 Agent RL 的算力空转消除掉**，从而能负担起"超长程任务（上千步、数小时）"的强化训练——这正是长程 Coding Agent 能力的来源。

### 4.3 "楼梯式优化"的训练路径

GLM-5.1 的能力提升被描述为**楼梯式优化**：不是一次到位，而是通过异步 RL 在真实任务中**一段段台阶式地爬升**——每一轮交互-反馈-更新都把"自主执行步数 / 任务复杂度"往上推一格，最终累积出 1700 步连续工具调用、8 小时自主工作的开源纪录。

## 5. 国产芯片全栈适配（工程落地）

GLM-5.1 发布即完成与多款国产芯片的全栈适配：**华为昇腾、摩尔线程、海光、寒武纪、昆仑芯、沐曦、燧原**。其中昇腾上做到 Layer 级 MoE 绝对均衡 + 吞吐 +30%（见 §3.2），摩尔线程等也同期完成适配。意义在于：**算法优化（MoE 均衡、稀疏注意力）与国产硬件的并行/调度方式绑定调优**，让超大模型能在非英伟达生态上高效跑起来。

## 6. 一页总结

```
四个矛盾 → 四套机制
   │
   ├── 理解 vs 生成：自回归填空（挖span + 自回归补全，一套目标统一NLU与生成）
   │
   ├── 大 vs 贵：MoE(754B总参稀疏激活) + DSA(动态筛关键token，砍1/2~2/3算力，长上下文不掉)
   │
   ├── 训练 vs 实战：异步Agent RL「Slime」(生成/训练解耦异步流水 → 支撑超长程任务训练)
   │                  → 楼梯式优化 → 1700步工具调用 / 8小时自主
   │
   └── 算法 vs 硬件：国产芯片全栈适配(昇腾Layer级MoE绝对均衡，吞吐+30%)
```

**一句话**：GLM-5.1 不是靠堆参数，而是靠 **"填空式统一理解与生成 + 稀疏注意力压成本 + 异步RL练长程自主 + 算法绑硬件调优"** 四套机制，把一个开源模型做成了能在复杂工程里自主干长任务、且能跑在国产卡上的 Coding Agent。
